Cuestionario de JavaScript

Pon a prueba tus conocimientos sobre conceptos fundamentales de JavaScript. Responde las 10 preguntas y descubre tu puntuación.

1. ¿Qué es JavaScript?

a) Un lenguaje de programación para el desarrollo backend
b) Un lenguaje de scripting para páginas web
c) Un framework para desarrollo frontend
d) Un lenguaje de base de datos

2. ¿Qué significa DOM?

a) Document Object Model
b) Data Object Management
c) Digital Orientation Module
d) Document Order Model

3. ¿Qué es una closure en JavaScript?

a) Una función que tiene acceso al ámbito de su función exterior
b) Una forma de ocultar variables
c) Un método para cerrar ventanas del navegador
d) Una función que se ejecuta automáticamente

4. ¿Qué método se utiliza para convertir un objeto a formato JSON?

a) JSON.parse()
b) JSON.stringify()
c) Object.toJSON()
d) JSON.encode()

5. ¿Qué es el hoisting en JavaScript?

a) El comportamiento de mover declaraciones al top del ámbito
b) Una forma de optimizar el código
c) Un método para elevar elementos en el DOM
d) Una técnica para manejar eventos

6. ¿Cuál es la diferencia entre "==" y "==="?

a) "==" compara valor y tipo, "===" solo valor
b) "===" compara valor y tipo, "==" solo valor
c) No hay diferencia
d) "==" es para números, "===" para strings

7. ¿Qué es una promesa en JavaScript?

a) Un objeto que representa la finalización o el fracaso de una operación asíncrona
b) Una función que se ejecuta después de un tiempo determinado
c) Una declaración de intenciones en el código
d) Un método para garantizar que una función se complete

8. ¿Qué hace el método "addEventListener"?

a) Permite escuchar eventos en elementos del DOM
b) Agrega una función a la cola de ejecución
c) Crea un nuevo evento personalizado
d) Registra un observador para cambios en el DOM

9. ¿Qué es el contexto de ejecución (execution context) en JavaScript?

a) El entorno en el que se ejecuta el código JavaScript
b) El ámbito de una función
c) El objeto window del navegador
d) El estado actual de la pila de llamadas

10. ¿Qué son las arrow functions?

a) Una sintaxis más corta para escribir funciones
b) Funciones que no tienen su propio "this"
c) Funciones que no pueden ser utilizadas como constructores
d) Todas las anteriores
